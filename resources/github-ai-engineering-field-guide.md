# AI Engineering Field Guide

## 基本信息

- 类型：GitHub 资料型仓库
- 链接：[alexeygrigorev/ai-engineering-field-guide](https://github.com/alexeygrigorev/ai-engineering-field-guide)
- 作者：Alexey Grigorev
- 仓库定位：面向 AI Engineering 角色、技能和面试的数据驱动 field guide
- 当前判断：这是值得长期跟踪的高质量参考项目，但现阶段更适合作为阅读型资源，而不是立即本地运行的代码项目

## 它解决什么问题

这个仓库的价值不在于教你跑某一个 demo，而在于帮你建立对 `AI Engineer` 这个岗位和能力结构的整体认知。仓库首页明确说明，它基于真实数据整理内容，包括实际 job descriptions、真实 interview experiences 和 practitioners 的真实故事。当前公开内容主要覆盖：

- AI Engineer 角色定义
- 技能与职责分析
- 面试流程与题型
- 学习路径
- 作品集项目思路
- 就业市场数据

适合解决的问题：

- AI Engineer 到底在做什么
- 这个方向的能力结构是什么
- 从学习到求职，大致应该怎么规划
- 后续做什么项目更接近真实招聘预期

## 为什么值得看

- 它不是泛泛而谈的入门教程，而是围绕真实岗位、真实面试和真实数据整理出来的研究型资料仓库
- 仓库首页明确强调它不是随意堆砌的 AI 内容，而是从真实数据里归纳模式
- 仓库首页明确给出了多个高价值主题：`role`、`interview`、`learning-paths`、`portfolio`、`job-market`
- 对这个学习仓库来说，它更像一张“行业地图”，能帮助我们决定后续讲义、实验和项目该往哪里长

从仓库首页当前信息看，比较值得注意的几类数据包括：

- `role/` 下的 responsibilities 和 use cases 分析
- `interview/` 下的 coding problems、case studies、home assignments、company-by-company data
- `job-market/` 下的 structured job descriptions

## 适合哪个阶段

- 当前阶段就适合开始阅读，但阅读目标不是“全部看完”，而是先拿它校准方向
- 最适合的阶段：
  - 仓库初始化阶段
  - 学习路径设计阶段
  - 项目选题阶段
  - 面试与能力复盘阶段

## 建议先看哪些部分

推荐优先顺序：

1. `README.md`
2. `role/`
3. `learning-paths/`
4. `portfolio/`
5. `interview/`
6. `job-market/`

这样看比较合适，因为：

- `README.md` 先帮你建立全局地图
- `role/` 先回答“AI Engineer 是什么”
- `learning-paths/` 帮助校准学习顺序
- `portfolio/` 帮助后续设计更像真实工作的项目
- `interview/` 和 `job-market/` 更适合在你有一定基础后再系统吸收

## 现阶段不必优先看的部分

下面这些内容现阶段不需要投入太多时间：

- 过细的求职细节
- 大量公司级面试差异
- 大规模岗位数据细节

原因不是它们不重要，而是当前阶段的首要目标仍然是跑通最小 Agent 学习闭环。

## 是否需要 clone 到本地

当前建议：`暂时不需要`

原因：

- 这个仓库当前对你的主要价值是认知框架和路径参考，不是本地执行代码
- 先通过网页阅读 `README` 和关键目录，通常就足够决定它对当前学习有没有帮助
- 现在 clone 到本地，大概率只是把资料从网页搬到本地，收益不高

## 什么时候值得 clone

只有在出现下面情况时，才建议把它拉到本地：

- 你准备系统拆解它的目录和 markdown 内容
- 你要把其中的学习路径、项目思路或岗位数据转成这个仓库里的讲义、练习或项目任务
- 你需要对内容做本地检索、批量整理或二次分析
- 你准备分析其中的结构化岗位数据或原始提取数据

如果只是日常参考，网页阅读通常就够了。

## 在本仓库里的使用方式

建议把它当成以下几类上游输入：

- `lectures/` 的方向参考
- `projects/` 的选题参考
- `reviews/` 的能力对照参考
- `resources/` 中的长期跟踪项目

不建议把它当成：

- 第一个需要本地复现的项目
- 当前阶段的主线教材

## 后续接入建议

当我们开始创建第一批讲义和项目时，可以从这个仓库里继续提炼：

- AI Engineer 能力地图
- 项目选题标准
- 学习阶段和阶段目标
- 作品集项目应该展示哪些能力

到那时，如果需要高频引用，再决定是否 clone 到本地。
