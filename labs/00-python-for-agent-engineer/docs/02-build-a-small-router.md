# Lab 02：写一个最小任务路由器

## 实验目标

练习函数拆分、条件分支和返回值，而不是把逻辑都堆进一个 `main()`。

## 运行方式

写一个脚本，输入一句任务描述，然后：

1. 先把它分类为 `data / io / general`
2. 再根据类型输出一条不同提示

## 关键代码

至少拆成两个函数：

- `classify_task()`
- `build_reply()`

## 预期输出

- 输入包含 `json` 时，走数据处理分支
- 输入包含 `file` 时，走文件处理分支
- 其他情况走默认分支

## 常见报错

- 分类函数直接 `print()`，后面无法组合
- 分支写死在 `main()`，改动困难
- 同一个函数既分类又输出又打印

## 实验结论

做完后你应该能解释：

- 为什么 `return` 比 `print` 更重要
- 什么时候该拆函数
- 为什么 workflow 本质上先是控制流问题
