# Lab 06：做一个最小模型调用器

## 实验目标

把前面所有 Python 基础接到第一次模型调用上。

## 运行方式

实现一个最小命令行程序：

1. 从命令行或固定变量读取输入
2. 调用一个 `call_model()` 函数
3. 打印结果
4. 对空输入和请求失败做最小处理

如果你暂时没有可用的模型 API，也可以先用 mock 实现把结构跑通，再替换成真实 SDK。

## 关键代码

- `main.py`
- `model_client.py`
- `config.py`
- `logger.py` 或最小日志封装

## 预期输出

- 输入有效时，返回一条模型响应
- 输入为空时，给出明确报错
- 调用失败时，至少知道错误属于哪一类

## 常见报错

- API 调用逻辑散落在多个文件里
- 配置、请求和输出混在一起
- 只有最后答案，没有运行记录

## 实验结论

做完后你应该能解释：

- 第一次模型调用里，哪些问题属于 Python 基础，哪些属于 LLM 应用
- 为什么应该立刻进入项目 1，而不是继续加新语法
