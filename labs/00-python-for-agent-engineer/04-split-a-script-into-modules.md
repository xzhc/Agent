# Lab 04：把一个脚本拆成多个模块

## 实验目标

练习模块边界、导入关系和最小类型标注。

## 运行方式

把你之前一个单文件脚本拆成至少三个文件：

- `main.py`
- `router.py`
- `utils.py`

如果你已经完成 lab 02 或 lab 03，可以直接拿那份代码来拆。

## 关键代码

- `main.py` 只负责串流程
- `router.py` 放核心业务函数
- `utils.py` 放小工具函数
- 给关键函数加基础类型标注

## 预期输出

- 代码能像拆分前一样运行
- 每个模块职责更清晰
- 你自己能用一句话描述每个文件的职责

## 常见报错

- 导入路径错误
- 模块之间相互依赖
- 文件拆开了，但职责还是混乱

## 实验结论

做完后你应该能解释：

- `main.py` 为什么不该装所有逻辑
- 模块拆分为什么是后续 Agent 项目可维护性的起点
