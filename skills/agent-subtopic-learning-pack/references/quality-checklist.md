# Quality Checklist

Run this checklist before finishing a learning-pack generation task.

## Structural integrity

- Does each top-level directory use the same topic root folder?
- Does each topic root have a `README.md`?
- Are links updated to the new topic-scoped paths?

## Workflow integrity

- Does the lecture root README clearly map lectures to labs, notes, projects, and reviews?
- Does the pack implement the repo's `lecture -> lab -> notes -> project -> review` flow?
- Are project gates placed after enough prerequisite learning stages?

## Scope control

- Does the pack follow the child roadmap instead of drifting into broader material?
- Does the selected mode match the user's requested depth?
- If the user deferred labs, did you avoid writing them in full?

## Practicality

- Can the learner tell what to do next after each lecture?
- Can the learner tell when to start a project?
- Can the learner tell where to record learning and where to review it?

## Repo fit

- Did you update only the necessary root READMEs?
- Did you preserve the repository's existing writing style?
- Did you avoid turning the pack into a resource list or abstract curriculum essay?
