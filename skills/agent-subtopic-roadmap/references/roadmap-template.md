# Roadmap Template

Use this template when creating a new child roadmap file.
Adapt the topic-specific content, but keep the structure stable unless the user asks for a shorter variant.

## Recommended file shape

```md
# <主题>学习路线图（面向 Agent Engineer）

## 目的

说明这个主题为什么值得学，以及它怎样服务后续 Agent 开发。

## 设计原则

- 说明这个主题的取舍方式
- 强调最小闭环、高频能力、工程导向

## 学习边界

### 现在优先学什么

- 高价值、高频、能直接支撑后续 Agent 开发的内容

### 现在不用展开学什么

- 暂时不会成为瓶颈的相邻内容
- 明确写出为什么先不学

## 阶段 0：<子阶段名>

### 目标

### 学什么

### 为什么这一步重要

### 典型产出

### 完成标志

## 阶段 1：<子阶段名>

### 目标

### 学什么

### 为什么这一步重要

### 典型产出

### 完成标志

## 推荐学习顺序

1. ...
2. ...

## 每一阶段都应该留下的仓库产出

- `lectures/`
- `labs/`
- `notes/`
- `projects/`
- `reviews/`

## 适合这个主题的阶段项目

### 项目 1：...

目标：

- ...

它对应的能力：

- ...

## 这一主题里的 Codex 协作主线

- ...

## 学完这个主题后，你应该具备什么能力

- ...

## 判断是否可以进入下一主题

- ...
```

## Topic translation rules

- Convert broad subject names into Agent-oriented outcomes.
- Prefer "what this enables later" over "what this topic historically includes".
- Use section titles that are stable across roadmap files.
- Keep each stage actionable enough that later lecture generation can target it directly.
