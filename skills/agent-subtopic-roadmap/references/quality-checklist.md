# Quality Checklist

Run this checklist mentally before finishing a child roadmap.

## Scope

- Is the topic clearly narrowed for Agent Engineer goals?
- Does the roadmap say what to learn later, not now?
- Would this document prevent future lecture generation from drifting into irrelevant content?

## Structure

- Does the file follow the same overall section pattern as other roadmap notes?
- Does each stage include `目标 / 学什么 / 为什么重要 / 典型产出 / 完成标志` or a justified equivalent?
- Does the roadmap end with a concrete progression gate into the next topic?

## Practicality

- Are there explicit project or lab outputs?
- Are the completion criteria observable rather than vague?
- Does the roadmap connect the subtopic to later Agent work such as model calls, tool use, evaluation, debugging, or system design?

## Repo integration

- Did you update `notes/roadmaps/01-agent-engineer-roadmap.md` if a parent bullet should now link to the child roadmap?
- Did you update `notes/README.md` if this child roadmap will likely be reused?
- Did you avoid unrelated file churn?
