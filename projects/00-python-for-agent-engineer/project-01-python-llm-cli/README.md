# Project 01：Python LLM CLI

## 项目目标

做一个最小但完整的命令行 LLM 调用器，把前 6 讲学到的 Python 基础串成一个可运行项目。

## 什么时候开始做

只有在完成下面内容后再开始：

- `lectures/00-python-for-agent-engineer/01` 到 `06`
- `labs/00-python-for-agent-engineer/01` 到 `06`
- `notes/00-python-for-agent-engineer/study-notes.md` 已经补到 lecture 06 和 lab 06

## 用户场景

用户在命令行输入一段请求，程序：

1. 读取配置
2. 调用模型
3. 打印结果
4. 留下最小运行记录
5. 对常见错误给出清晰反馈

## 架构说明

建议至少拆成这些模块：

- `main.py`：解析输入，串主流程
- `config.py`：加载环境变量或配置
- `model_client.py`：封装模型调用
- `logger.py`：记录关键运行信息
- `storage.py`：把本次运行写入本地文件或 JSONL

## 必做能力点

- 环境变量读取
- 函数拆分
- 模块组织
- JSON 或文件写入
- 最小错误分类
- 最小日志

## 运行方法

你需要自己在项目目录里写运行说明，但至少应包括：

- 如何创建环境
- 如何安装依赖
- 如何设置配置
- 如何运行示例输入

## 评估方法

至少验证下面四种情况：

1. 正常输入
2. 空输入
3. 配置缺失
4. 模型调用失败或 mock 失败

## 已知问题

在项目 README 里主动写出：

- 现在还没有做哪些健壮性处理
- 哪些错误只是最小处理
- 哪些地方以后进入 Agent 阶段还需要重构

## 后续迭代方向

- 加结构化输出
- 加更明确的错误分类
- 加简单测试
- 为后续 tool calling 保留接口

## 项目完成标志

- 你能向别人解释每个模块负责什么
- 你能说清楚一次调用失败应该去哪里查
- 你知道这还不是 Agent，但已经是 Agent 的基础
